Prioritized Remediation Plan

1
Consolidate signup_completed to one call site call_site_c loses 3 users entirely (NULL user_id). call_site_b omits plan and source. Move all signup_completed firing to the backend, post-authentication. Estimated impact: recover 3 lost conversions, gain consistent properties on all 39 events.
2
Fix UTM parameter persistence 35% of sessions have no attribution data. Implement server-side UTM capture at the first redirect hop and persist in a first-party cookie. Audit ad platform destination URLs for UTM stripping. Estimated impact: reduce NULL-UTM rate from 35% to under 10%.
3
Reconcile session creation (client vs. server) 6 orphan sessions have events but no session record. Investigate the race condition between Amplitude's client SDK and the server-side session writer. Add a dbt relationships test with a warn threshold. Estimated impact: 6 conversions become attributable.
4
Enforce campaign naming convention Deploy a UTM builder tool for the marketing team. Normalize existing data in the dbt staging layer with LOWER() and REPLACE(). Add an accepted_values test on utm_campaign. Estimated impact: eliminates 3-way campaign fragmentation in all dashboards.
5
Deduplicate conversions at the source Backend retry logic should be idempotent. If signup_completed fires twice for the same user (as with usr_136 and usr_143), the conversions table should not insert a second row. Add a unique constraint on user_id. Interim fix: dedup in dbt staging.
